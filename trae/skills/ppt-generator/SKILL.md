# PPT生成器 SKILL

## 功能描述

本SKILL用于根据用户需求自动生成符合公司视觉规范的PPT，支持封面、目录、内容和结尾页的生成。

## 指令格式

```markdown
# PPT生成

## 基本信息
- 标题: [PPT标题]
- 副标题: [PPT副标题]
- 日期: [日期，格式：YYYY.MM]

## 目录
1. [第一部分标题]
2. [第二部分标题]
3. [第三部分标题]

## 内容

### 第一部分 [第一部分标题]
[第一部分内容]

### 第二部分 [第二部分标题]
[第二部分内容]

### 第三部分 [第三部分标题]
[第三部分内容]

## 结尾
[结尾内容]
```

## 使用示例

```markdown
# PPT生成

## 基本信息
- 标题: 2024年公司年度报告
- 副标题: 创新驱动，合作共赢
- 日期: 2024.01

## 目录
1. 公司概况
2. 业务发展
3. 未来规划

## 内容

### 第一部分 公司概况
公司成立于2010年，是一家专注于科技创新的企业，拥有员工500余人，业务覆盖全国。

### 第二部分 业务发展
2023年公司业务实现了30%的增长，主要业务包括人工智能、云计算和大数据分析。

### 第三部分 未来规划
未来三年，公司计划在海外市场拓展业务，加大研发投入，提升核心竞争力。

## 结尾
感谢各位的聆听，期待与大家的合作！
```

## 实现逻辑

1. 解析用户输入的Markdown指令
2. 提取PPT的基本信息、目录和内容
3. 生成符合公司视觉规范的HTML PPT
4. 提供HTML到PPTX的转换建议

## 图表规范

### 【AI强制执行规则】
1. 严格遵循本文件所有图表规范，包括样式、布局、颜色和数据标注。
2. 强制使用开源图表库 ECharts (Apache 2.0 许可证)。
3. 坐标轴文字颜色必须使用 #999999。

### 一、图表库要求
- 强制使用: 开源图表库 ECharts (Apache 2.0 许可证)。
- 引入方式: 通过官方CDN引入，确保版本最新。

### 二、样式规范

#### 1. 颜色规范
- 图表主色: 使用官方主色 #00B4FF。
- 辅助色: 依次使用 #8254F0, #28E6E6, #0082FF, #00F0BE, #FF6E00, #94B5F5, #8F61D1, #DCA1E4, #FFC60A。
- 图表标题颜色: 使用官方字体通用色 #2E2E2E。
- 坐标轴文字颜色: 强制使用 #999999。
- 图例文字颜色: 使用官方字体通用色 #2E2E2E。

#### 2. 字体规范
- 图表标题: 方正兰亭中粗黑简体，24px。
- 坐标轴标签: 方正兰亭细黑简体，14px。
- 图例文字: 方正兰亭细黑简体，14px。

#### 3. 标题位置
- 图表标题统一放置在图表左上角，距左 40px，距顶 30px。

### 三、布局规范（详细建议）

#### 1. 图表尺寸
- 宽度: 占页面内容区域宽度的 50%-70%。
- 高度: 固定为 350px。

#### 2. 常见布局
- 左图右文:
  - 图表居左，宽度占页面内容区域的 55%。
  - 右侧放置文字说明，宽度占页面内容区域的 40%。
  - 图表与文字之间保持 40px 的间距。
- 右图左文:
  - 图表居右，宽度占页面内容区域的 55%。
  - 左侧放置文字说明，宽度占页面内容区域的 40%。
  - 图表与文字之间保持 40px 的间距。
- 上图下文:
  - 图表居上，高度占页面内容区域的 60%。
  - 下方放置文字说明，高度占页面内容区域的 35%。
  - 图表与文字之间保持 20px 的间距。

#### 3. 间距规范
- 图表与页面边缘: 图表距离页面左/右/上/下边缘至少 44px。
- 图表与文字: 图表与旁边的文字块之间保持 20px - 40px 的间距。

### 四、数据标注要求
- 强制标注: 所有图表必须在右下角或下方清晰标注数据来源。
- 标注格式: "数据来源：[来源名称] | [年份]"。
- 字体: 方正兰亭细黑简体，12px，颜色 #888888。

### 五、图表类型推荐
- 趋势分析: 使用折线图。
- 数据对比: 使用柱状图。
- 构成分析: 使用饼图或环形图。
- 分布分析: 使用散点图或热力图。

## 图标使用规范

### 图标整体搭配比例（强制）
1. **Web Awesome 官方图标：占比 70%**
2. **系统原生 Emoji 图标：占比 30%**
3. 所有页面必须遵守该比例，禁止只用一种图标。

### Web Awesome 图标规则（70%）
- 强制使用库：Web Awesome（wa-icon）
- 必须引入 CDN：
  ```html
  <link rel="stylesheet" href="https://ka-f.webawesome.com/webawesome@3.5.0/styles/themes/default.css">
  <link rel="stylesheet" href="https://ka-f.webawesome.com/webawesome@3.5.0/styles/native.css">
  <script type="module" src="https://ka-f.webawesome.com/webawesome@3.5.0/webawesome.loader.js"></script>
  ```
- 使用标签：`<wa-icon name="图标名"></wa-icon>`
- 适用场景：核心功能、业务图标、流程、模块、标题区、数据区
- 颜色：必须使用规范色 #00B4FF / #8254F0 / #2E2E2E / #999999
- 尺寸：标题区 24–32px，正文区 18–24px

### Emoji 图标规则（30%）
- 仅使用系统原生 Emoji，禁止图片化
- 适用场景：点缀、情绪表达、分类标签、轻量提示
- 禁止用于：核心功能图标、数据图标、标题主图标
- 大小：与正文保持一致，不放大变形
- 颜色：保持 Emoji 原生色彩，不做修改

### 场景分配规则（AI 必须遵守）
- 标题模块、功能模块、数据图表、流程步骤 → 用 Web Awesome
- 分类标签、小标注、情绪强调、列表点缀 → 用 Emoji
- 一页 PPT 里：10 个图标 → 7 个 Web Awesome + 3 个 Emoji

## 矢量插画规范

### 1. 插画来源
- 统一引用 unDraw 矢量插画。

### 2. 插画主色
- 统一设置为品牌主色 #00B4FF。

### 3. 格式要求
- 使用 SVG 矢量格式，清晰可缩放。

### 4. 应用场景
- 仅用于内容页配图。

### 5. 使用规则
- 单页 PPT 最多使用 6 张插画
- 插画尺寸统一、风格统一
- 不遮挡文字、不破坏版式结构

### 6. 实现建议
- **内联SVG**：为确保插画显示稳定，建议使用内联SVG格式，避免依赖外部CDN链接
- **颜色设置**：使用CSS滤镜或直接在SVG中设置颜色为品牌主色 #00B4FF
- **尺寸控制**：保持插画尺寸在150x150px左右，确保页面布局美观
- **背景色**：可使用规范中的背景色（如 #E6F8FF, #F1ECFC, #FBEDE2）增强视觉效果
- **示例代码**：
  ```html
  <div style="width: 150px; height: 150px; background-color: #E6F8FF; display: flex; align-items: center; justify-content: center; border-radius: 8px;">
    <svg width="100" height="100" viewBox="0 0 100 100" fill="none" xmlns="http://www.w3.org/2000/svg">
      <circle cx="50" cy="50" r="40" fill="#00B4FF" opacity="0.7"/>
      <path d="M50 30L60 45H40L50 30Z" fill="white"/>
    </svg>
  </div>
  ```

## 分割线规范

### 1. 分割线类型与样式
AI 必须严格区分以下三种分割线样式，不得混用：

| 类型 | 代码规范 (CSS) | 适用场景 |
| :--- | :--- | :--- |
| 默认分割线 | `border: 1px solid #DCDDE5;` | 常规内容区、模块间距 |
| 深色分割线 | `border: 1px solid #B0B2BF;` | 深色背景模块、强调区 |
| 虚线分割线 | `border: 1px dashed #B0B2BF;` | 流程步骤、待办状态 |

### 2. 渐变分割线（强制规范）
- **渐变公式**：`linear-gradient(90deg, #0082FF 0%, #8254F0 100%)`
- **粗细**：固定为 1px
- **适用场景**：
  - 关键数据区块顶部 / 底部
  - 渐变标题栏下方的分隔
  - 强调模块的边缘装饰

### 3. 通用规则（AI 必执行）
- **尺寸统一**：所有分割线粗细必须为 1px，禁止 2px 或其他尺寸
- **位置规范**：分割线通常置于模块底部或模块之间
- **间距**：分割线上下需保留 16px - 24px 的安全间距，不得紧贴文字或图片

## 布局决策规则

### 角色定义
你是一个"PPT布局决策引擎"，你的唯一职责是：
👉 根据内容结构，选择最合适的页面布局

你不负责：
- 不拆分页
- 不删减内容
- 不改写内容

你只做一件事：
👉 判断"这一页应该用什么布局"

### 一、输入内容分析（必须执行）

你必须先对内容进行结构解析，并识别：

1. 内容数量（信息块数量）
2. 是否存在：
   - 多条要点（列表）
   - 数据/数字
   - 对比关系（A vs B）
   - 步骤/流程
   - 分类结构（多个模块）
   - 单一核心结论

### 二、布局类型（只允许从以下选择）

你只能选择以下布局之一：

【1】左图右文  
【2】右图左文  
【3】上图下文  
【4】双列布局  
【5】三列卡片布局  
【6】对比布局  
【7】流程布局  
【8】框架结构布局  
【9】强调布局（单核心）  
【10】数字强调布局  

禁止自创布局。

### 三、布局选择逻辑（强制规则）

按优先级从上到下判断：

#### ① 强优先级（直接决定）

如果满足以下任一条件，直接选定布局：

- 出现"步骤 / 阶段 / 流程"
  → 使用【流程布局】

- 出现"对比 / A vs B / 优劣"
  → 使用【对比布局】

- 出现"结构 / 架构 / 模块关系"
  → 使用【框架结构布局】

- 出现"单一强结论 / 标语 / 金句"
  → 使用【强调布局】

- 出现"核心数字 / KPI / 增长数据"
  → 使用【数字强调布局】

#### ② 内容结构判断

如果没有命中上面规则：

- 信息块 = 2
  → 使用【双列布局】

- 信息块 = 3
  → 使用【三列卡片布局】

- 信息块 ≥ 4
  → 使用【列表型（左文结构）】（归类为左图右文 or 无图结构）

#### ③ 图文判断（最后兜底）

如果内容中明确包含：
- 图片 / 插画 / 图表

则：

- 图是辅助说明
  → 使用【左图右文】

- 图是核心内容（如数据图）
  → 使用【上图下文】

### 四、增强决策（强调页判断）

在不改变原内容的前提下：

如果检测到以下情况：

- 出现总结性语句（如：因此 / 核心是 / 本质是）
- 出现强观点表达
- 出现可独立成句的短内容

👉 标记：建议增加【强调布局页】

⚠️ 注意：
只做"建议"，不强制拆页

### 五、输出格式（必须遵守）

你必须输出以下结构：

```json 
{
  "layout": "选中的布局名称",
  "reason": "为什么选择该布局（简要说明）",
  "structure": "推荐结构描述（如：左右50% / 三列 / 上下结构）",
  "emphasis_suggestion": "是否建议增加强调页（yes/no + 原因）"
}
```
